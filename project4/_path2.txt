React app runs in the browser, not inside cluster


So when the browser executes:
POST http://django-svc:8000/api/v1/register/


it tries to resolve:
django-svc  ❌ (from YOUR laptop / browser)


✔️ Use Nginx as Reverse Proxy


1. Change frontend API base URL
In React Docker File:
ENV VITE_SERVER_BASE_URL=/api/v1


2. Update your Nginx config
server {
    listen 80;

    root /usr/share/nginx/html;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api/v1 {
        proxy_pass http://django-svc:8000/api/v1;
    }
}

3. Flow becomes

Browser
   ↓
react-svc (NodePort)
   ↓
Nginx (inside container)
   ↓
django-svc (ClusterIP)
   ↓
Django pods




Browser → only runs HTML, CSS, JS
Nginx → runs inside your React container (pod) in Kubernetes


🔥 Real flow (clarified)

Browser sends request:
GET http://localhost/api/v1/register/

That hits:
react-svc (NodePort) → React pod → Nginx inside container

Nginx sees:
/api/v1/

and applies:
proxy_pass http://django-svc:8000/api/v1/;

Now Nginx (inside the cluster) makes a new request:
http://django-svc:8000/api/v1/register/
